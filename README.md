# NixOS Configuration

Flake-based NixOS config for a personal dev machine. Clone to `~/nixos-config/` and run
`sudo nixos-rebuild switch` to apply.

## Quick Start (new machine)

```bash
git clone <repo-url> ~/nixos-config
cd ~/nixos-config
sudo nixos-rebuild switch
```

> You'll also need to copy `hardware-configuration.nix` from the new machine's `/etc/nixos/`
> since it's hardware-specific and not portable.

---

## Common Commands

### Rebuild / Apply Changes

```bash
# Apply config changes (most common)
sudo nixos-rebuild switch

# Test config without making it the default boot entry
sudo nixos-rebuild test

# Build but don't switch (dry run)
sudo nixos-rebuild build
```

### Updating

```bash
# Update all flake inputs (nixpkgs, home-manager, stylix, etc.)
nix flake update

# Update a single input
nix flake update nixpkgs

# After updating, rebuild to apply
sudo nixos-rebuild switch
```

### Searching for Packages

```bash
# Search nixpkgs
nix search nixpkgs <name>

# Temporarily run a package without installing
nix shell nixpkgs#<name>

# Check what options a NixOS module exposes
nixos-option <option.path>
```

## How to Install Software

### System-wide (all users)
Add to `environment.systemPackages` in `configuration.nix`:
```nix
environment.systemPackages = with pkgs; [
  package-name
];
```

### User-level (preferred for personal tools)
Add to `home.packages` in `home-manager/home.nix`:
```nix
home.packages = with pkgs; [
  package-name
];
```

### With a Home Manager module (`programs.<name>`)
Some programs have dedicated Home Manager modules that configure the program
*and* install it with sensible defaults (e.g. neovim, git, kitty, waybar).

**To check if a module exists:**
- Search online: https://home-manager-options.extendednix.dev/
- Or locally: `man home-configuration.nix | grep programs.<name>`

If `programs.<name>` shows up, use:
```nix
programs.<name> = {
  enable = true;
  # extra options here
};
```

If not, fall back to `home.packages`.

### When nixpkgs doesn't have the package at all
Options (in order of NixOS best-practice preference):
1. **Add the project's own flake** as an input in `flake.nix` (if they publish one)
2. **Write a custom derivation** (for simple cases — fetch binary, wrap it)
3. **Install outside Nix** (npm, cargo, pipx, etc.) and document it here

## Architecture

| File | Purpose |
|------|---------|
| `flake.nix` | Flake inputs/outputs; wires NixOS + Home Manager + Stylix |
| `configuration.nix` | System-level config (services, system packages, users) |
| `hardware-configuration.nix` | Auto-generated hardware config — **machine-specific** |
| `home-manager/home.nix` | User-level config (user packages, program settings) |
| `home-manager/theme.nix` | Stylix theming (Rose Pine colorscheme) |
| `home-manager/rofi.nix` | Rofi launcher config |

### Flake inputs (pinned in `flake.lock`)
- `nixpkgs` → `nixos-25.05`
- `home-manager` → `release-25.05`
- `stylix` → `release-25.05`
- `swww` → wallpaper daemon (pulled direct from upstream flake)

## Notes

### Stylix
It appears that Stylix needs to be enabled in both `configuration.nix` and `home-manager` (e.g. `home.nix`)
because that comes up often. When I just had it in `home.nix` it failed to build, but having it just in
`configuration.nix` seems to not allow overriding of anything `home-manager` might theme.

*Agent note: Currently the system-level module (`stylix.nixosModules.stylix`) is commented out in `flake.nix`
and theming runs only through Home Manager — this was working at last rebuild.*

The config sets `home-manager.backupFileExtension = "backup"` so Stylix can overwrite existing
dotfiles without failing.

### Hardware Configuration
`hardware-configuration.nix` is auto-generated by the NixOS installer and is machine-specific.
When setting up a new machine:
1. Install NixOS normally
2. Replace `hardware-configuration.nix` with the one from `/etc/nixos/hardware-configuration.nix`
3. Clone this repo and run `sudo nixos-rebuild switch`

### Kanata
Key remapping is handled by the `kanata` service configured inline in `configuration.nix`.
Currently remaps: Caps Lock → Escape.
